# Process

1. Run validation checks before commit.
2. Use semantic commit prefixes (`feat`, `fix`, `docs`, `chore`).
3. Open PR with problem, solution, and test evidence.
4. Address findings by severity.

# Verification

- CI checks pass.
- PR template is fully populated.
